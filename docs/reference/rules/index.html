greedy_welfare(instance: ~pabutools.election.instance.Instance, profile: ~pabutools.election.profile.profile.Profile | ~pabutools.election.profile.profile.MultiProfile, sat_class: type[pabutools.election.satisfaction.satisfactionmeasure.SatisfactionMeasure] = None, sat_profile: ~pabutools.election.satisfaction.satisfactionprofile.SatisfactionProfile | ~pabutools.election.satisfaction.satisfactionprofile.SatisfactionMultiProfile = None, is_sat_additive: bool = False, tie_breaking: ~pabutools.tiebreaking.TieBreakingRule = <pabutools.tiebreaking.TieBreakingRule object>, resoluteness: bool = True, initial_budget_allocation: ~collections.abc.Iterable[~pabutools.election.instance.Project] = None) → Iterable[Project] | Iterable[Iterable[Project]][source]#

General greedy scheme for approval profiles. It selects projects in rounds, each time selecting a project that lead to the highest increase in total satisfaction divided by the cost of the project. Projects that would lead to a violation of the budget constraint are skipped. :param instance: The instance. :type instance: pabutools.election.instance.Instance :param profile: The profile. :type profile: pabutools.instance.profile.ApprovalProfile :param sat_class: The class defining the satisfaction function used to measure the social welfare. It should be a class

inhereting from pabutools.instance.satisfaction.Satisfaction. If no satisfaction is provided, a satisfaction profile needs to be provided. If a satisfation profile is provided, the satisfaction argument is disregarded.

Parameters:

  • sat_profile (pabutools.instance.satisfaction.SatisfactionFunction) – The satisfaction profile corresponding to the instance and the profile. If no satisfaction profile is provided, but a satisfaction function is, the former is computed from the latter.

  • is_sat_additive (bool) – A boolean indicating if the satisfaction function is additive. This is directly deducted if sat_class is provided.

  • tie_breaking (pabutools.rules.tiebreaking.TieBreakingRule, optional) – The tie-breaking rule used. Defaults to lexico_tie_breaking.

  • resoluteness (bool, optional) – Set to False to obtain an irresolute outcome, where all tied budget allocations are returned. Defaults to True.

  • initial_budget_allocation (collection of Project, optional) – A potential initial budget allocation.

Return type:

set of Project
